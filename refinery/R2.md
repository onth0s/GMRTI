**R2 — How to Implement Syntactic Edge-Forcing Without Ithkuil's Overhead?** `[OPEN]`

Section 2g introduces Syntactic Edge-Forcing as a transmission fidelity mechanism in which compositional grammatical structure compels the receiver to traverse a specific relational edge as a constitutive condition of decoding. The mechanism is well-attested in formal languages (mathematical notation, root compounding) and is theoretically distinct from both Polyphonic Encoding and JIT-OM.

The design challenge is implementing a version that achieves meaningful entropy reduction without the prohibitive formulation overhead that makes Ithkuil practically unlearnable and Lojban practically unused. The optimization target is explicit: maximum entropy reduction per unit of formulation overhead. Ithkuil maximizes entropy reduction and ignores overhead; natural language minimizes overhead and ignores entropy reduction. The GMRTI's formal layer must find the useful middle.

The Concatenation Insight (2g.4) suggests one mechanism: requiring every formally expressed concept to declare its *primary relational operator* — a single mandatory edge that anchors the concept in semantic space and provides the receiver a traversal starting point. This is not a full compositional grammar; it is a minimal edge-declaration requirement grafted onto otherwise natural language expression. The Edge Taxonomy (2a.5) now provides the vocabulary for that declaration: a concept expressed as `A →[causal] B` rather than merely `A + B` forces not only the edge traversal but the edge-type, preemptively preventing Edge-Type Misattribution (2c.3).

**Proposed Minimal Prototype (v0.9):** We mandate an explicit primary relational operator in key structural claims. For example, instead of writing "GMRTI reduces semantic entropy", the prototype requires "GMRTI [->causal] reduces semantic entropy". This forces edge-typing without requiring a fully novel grammar. We propose testing this in "self-annotation mode" first (Self-to-Future-Self transmission) as a low-risk validation sandbox before applying it to community dialogue.

**Open questions:** Is the primary relational operator approach sufficient, or does it only push the entropy problem back one level (onto the shared vocabulary of operators and edge-types)? What set of primitive relational operators is both small enough to learn and expressive enough to cover the concept-types where edge-forcing adds the most value — and how does this set relate to the Edge Taxonomy's eight types? What is the target use case for first implementation — constitutional drafting (community-to-community), GMRTI-mediated dialogue (self-to-other), or self-annotation (self-to-future-self)? The formal layer looks structurally different depending on the primary transmission mode it serves, and attempting to serve all three simultaneously in a first implementation risks designing a system that serves none well. Should the first implementation begin with a single transmission mode and generalize from demonstrated success?
